Quick Start

OpenCode connecté a GitHub Copilot en 3 minutes. Support natif, pas besoin de proxy.

C'est quoi OpenCode ?

OpenCode est un agent IA de coding en terminal, open source (MIT), avec 120K+ stars sur GitHub. C'est une alternative a Claude Code qui supporté 75+ providers dont GitHub Copilot nativement.

🌱
Open Source (MIT)
Code source ouvert, pas de vendor lock-in, communauté active.
🔗
Copilot natif
Support GitHub Copilot intégré. Pas de proxy, pas de hack.
⚙
75+ providers
OpenAI, Anthropic, Google, Ollama, Azure, Bedrock, et plus.
💻
TUI moderne
Interface terminal Bubble Tea avec keybindings vim.
Avantage vs Claude Code + proxy

OpenCode supporté GitHub Copilot nativement. Pas besoin d'installer un proxy, pas de hack, pas d'API non-officielle. C'est un partenariat officiel entre GitHub et OpenCode (janvier 2026).

⚠ Consommation de tokens Copilot

Comme tout agent IA en terminal, OpenCode consomme beaucoup plus de tokens qu'un simple chat Copilot. Une session de travail peut consommer l'equivalent d'une journee de Copilot Chat. Voir la section Consommation de tokens pour les details.

Pre-requis

  • Node.js >= 18 (pour l'install npm) ou Homebrew / curl
  • GitHub Copilot actif (Individual, Pro, Pro+, Business, ou Enterprise)
node --version    # >= 18
npm --version

Installation

# npm (recommandé)
npm install -g opencode-ai@latest

# ou bun
bun install -g opencode-ai@latest

# Vérifiez
opencode --version
Autres méthodes d'installation
# Homebrew (macOS/Linux)
brew install anomalyco/tap/opencode

# Scoop (Windows)
scoop install opencode

# curl
curl -fsSL https://opencode.ai/install | bash

Connexion a GitHub Copilot

Lancez OpenCode dans votre projet
cd votre-projet
opencode
Connectez Copilot

Dans l'interface OpenCode, tapez :

/connect

Sélectionnez "GitHub Copilot" dans la liste.

Authentifiez-vous

Un code s'affiche. Ouvrez github.com/login/device et entrez le code.

Choisissez votre modèle
/models

Sélectionnez un modèle Copilot (ex: copilot.gpt-4.1, copilot.claude-sonnet-4).

✓ C'est prêt !

OpenCode est connecté a Copilot. Tapez vos commandes directement.

Utilisation de base

# Lancer OpenCode
opencode

# Mode non-interactif (scripting)
opencode -p "explique ce fichier" -f json

# Raccourcis dans l'interface
# Ctrl+O : changer de modèle
# Ctrl+K : palette de commandes
# Ctrl+N : nouvelle session
# Ctrl+X : annuler la generation

Detection automatique du token

Si vous utilisez déjà Copilot dans VS Code, le GitHub CLI, ou Neovim, OpenCode détecté automatiquement votre token depuis :

  • ~/.config/github-copilot/hosts.json
  • ~/.config/github-copilot/apps.json

Vous pouvez aussi fournir le token manuellement :

# Via variable d'environnement
export GITHUB_TOKEN=ghp_xxxxxxxxxxxx
opencode

Ou dans .opencode.json :

{
  "provider": {
    "copilot": {
      "apiKey": "ghp_xxxxxxxxxxxx",
      "disabled": false
    }
  }
}

Vérifier les providers authentifies :

opencode auth list

Consommation de tokens Copilot

⚠ OpenCode consomme votre quota Copilot

Chaque interaction avec OpenCode consomme des tokens de votre licence GitHub Copilot. Un agent terminal consomme beaucoup plus qu'un chat Copilot classique.

Pourquoi la consommation est elevee

  • Contexte volumineux : OpenCode envoie le contenu des fichiers, l'historique, les resultats d'outils a chaque requête (10 000 - 50 000 tokens d'entrée)
  • Appels en cascade : une commande peut generer 3 a 15 appels API (lecture, reflexion, edition, verification)
  • Reponses longues : les reponses contenant du code sont 5 a 20x plus longues qu'un chat

Ordres de grandeur

ActionTokens estimesEquivalent Copilot Chat
Question simple5 000 - 15 0003-5 messages chat
Edition de fichier50 000 - 150 00030-50 messages chat
Tache complexe200 000 - 500 000100-200 messages chat
Session 30 min500 000 - 2 000 0001 journee de chat

Comment reduire la consommation

  • Activez autoCompact (actif par défaut) : compresse le contexte a 95% d'utilisation
  • Choisissez des modèles économes : gpt-4o-mini ou gemini-2.0-flash pour les tâches simples
  • Soyez precis : "corrige la ligne 42 de server.js" consomme moins que "corrige les bugs"
  • Fermez et relancez regulierement pour reinitialiser le contexte
  • Mode non-interactif pour les tâches ponctuelles : opencode -p "question"

Fichier .opencode.json

OpenCode cherche la config dans cet ordre :

  1. ./.opencode.json (racine du projet, prioritaire)
  2. $XDG_CONFIG_HOME/opencode/.opencode.json
  3. $HOME/.opencode.json (global)

Agents : coder, task, title

OpenCode utilise 3 agents avec des rôles différents :

AgentRoleModèle recommandé
coderAgent principal : ecrit et edite le codecopilot.gpt-4.1 ou copilot.claude-sonnet-4
taskSous-tâches, recherche, reflexioncopilot.gpt-4o

Strategie d'economie de tokens

Utilisez un modèle puissant pour coder et des modèles légers pour task et title. Ainsi seules les tâches importantes consomment beaucoup.

    "copilot": { "disabled": false },

      "name": "Ollama (local)",

        "baseURL": "http://localhost:11434/v1"
      },
      "models": {
        "llama3": { "name": "Llama 3" },
        "mistral": { "name": "Mistral" }
      }

    "lmstudio": {

      "name": "LM Studio",
      "options": {
        "baseURL": "http://127.0.0.1:1234/v1"
      },
      "models": {
        "local-model": { "name": "Mon modèle local" }
      }
    }
  }
}

Recommandation pour l'entreprise

Si votre entreprise autorisé uniquement GitHub Copilot, OpenCode est le choix le plus sur : support officiel, pas de proxy hack, pas de risque de suspension de compte. Claude Code via proxy reste une option mais avec des risques.

Sécurité et risques

Points positifs

  • Open source : code auditable, pas de boite noire
  • Partenariat officiel GitHub : usage autorisé, pas d'API non-officielle
  • Privacy-first : OpenCode ne stocke pas votre code
  • Tokens locaux : stockes dans ~/.config/github-copilot/

Points de vigilance

  • Consommation de quota : peut epuiser le quota Copilot rapidement
  • Données transitent par les serveurs du provider (GitHub/OpenAI/Anthropic/Google)
  • Ne commitez jamais les tokens : ajoutez .opencode.json a .gitignore s'il contient des clés
  • Vérifiez la conformite avec votre equipe sécurité
