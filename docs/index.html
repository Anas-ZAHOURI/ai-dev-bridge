Gardez vos outils, utilisez votre abonnement

Votre entreprise vous fournit GitHub Copilot ou Gemini, mais vous travaillez mieux avec Claude Code ou OpenCode. Bonne nouvelle : les deux sont compatibles. Ce guide vous montre comment les brancher ensemble, sans rien casser, sans contourner les règles.

Le problème que tout le monde connaît

Vous avez vos habitudes, vos réflexes, votre productivité. Puis l'entreprise impose un outil différent.

❌ Abonnement imposé, outil limité

L'entreprise paye GitHub Copilot ou Gemini. Vous n'avez pas le choix de l'abonnement, mais l'interface chat ou l'autocompletion ne couvrent pas vos besoins.

❌ Perte de réflexes et de vitesse

Vous avez appris à être productif avec Claude Code ou OpenCode. Changer d'outil, c'est perdre des semaines de muscle memory et de workflows rodées.

✅ Un proxy local, transparent

Un petit serveur local traduit les appels entre votre outil préféré et l'API de l'abonnement entreprise. Rien n'est installé sur les serveurs de l'entreprise.

✅ Conforme aux standards

Le trafic passe par l'abonnement officiel. Pas de clé API personnelle, pas de compte parallèle. Vous restez dans le cadre autorisé par votre organisation.

Comment ça marche

Claude Code / OpenCode → Proxy local → GitHub Copilot / Gemini

Le proxy tourne sur votre machine, traduit le format d'API à la volée. Aucune donnée ne sort du circuit officiel.

Respectueux des standards entreprise

Cette approche ne contourne rien. Elle utilise l'abonnement que votre entreprise paye déjà, via les API officielles, en passant par le réseau autorisé. Vous changez l'interface, pas le tuyau.

✓ Abonnement entreprise utilisé ✓ Aucune clé API externe ✓ Proxy local uniquement ✓ Données dans le circuit officiel
