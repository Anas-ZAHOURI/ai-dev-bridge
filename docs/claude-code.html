Quick Start

Claude Code connecté a votre licence GitHub Copilot en 5 minutes.

⚠ Avertissement : consommation de tokens Copilot

Cette méthode utilise votre quota de tokens GitHub Copilot. Claude Code est extrêmement gourmand : il consomme 10 a 100 fois plus de tokens qu'un chat Copilot classique.

  • Une seule commande "edite ce fichier" = 50 000 a 150 000 tokens
  • Une session de 30 minutes = l'equivalent d'une journee entiere de Copilot Chat
  • Plusieurs utilisateurs rapportent avoir epuise leur quota mensuel en quelques jours

Voir la section Consommation de tokens pour les details et les strategies d'optimisation.

Pre-requis

node --version    # >= 18 requis
npm --version     # doit fonctionner
Node.js pas installé ?
# Via nvm (recommandé)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 20 && nvm use 20

# Ou téléchargez sur https://nodejs.org

Étape 1 : Installer Claude Code

npm install -g @anthropic-ai/claude-code
claude --version

Étape 2 : Lancer le proxy

Ouvrez un terminal :

npx copilot-api@latest start --claude-code
Le proxy démarre sur le port 4141
Un code s'affiche

Entrez ce code sur GitHub : XXXX-XXXX

Votre navigateur s'ouvre sur github.com/login/device

Entrez le code, autorisez l'application.

Choisissez vos modèles

Modèle principal (ex: gpt-4.1) et modèle rapide (ex: gpt-4.1-mini).

Commande copiée dans le presse-papier

Étape 3 : Lancer Claude Code

Ouvrez un 2e terminal et collez la commande. Elle ressemble à :

ANTHROPIC_BASE_URL=http://localhost:4141 \
ANTHROPIC_AUTH_TOKEN=dummy \
ANTHROPIC_MODEL=gpt-4.1 \
ANTHROPIC_SMALL_FAST_MODEL=gpt-4.1-mini \
DISABLE_NON_ESSENTIAL_MODEL_CALLS=1 \
claude
✓ C'est prêt !
Claude Code est connecté a Copilot. Tapez vos commandes normalement.

Configuration permanente

Pour ne plus taper les variables a chaque fois, créez ~/.claude/settings.json :

{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:4141",
    "ANTHROPIC_AUTH_TOKEN": "dummy",
    "ANTHROPIC_MODEL": "gpt-4.1",
    "ANTHROPIC_SMALL_FAST_MODEL": "gpt-4.1-mini",
    "DISABLE_NON_ESSENTIAL_MODEL_CALLS": "1",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  },
  "permissions": {
    "deny": ["WebSearch"]
  }
}

Ensuite, il suffit de :

# Terminal 1
npx copilot-api@latest start

# Terminal 2
claude

Comment ça marche

Claude Code parle le format Anthropic Messages API. GitHub Copilot parle le format OpenAI. Un proxy local traduit entre les deux.

Claude Code ───> Proxy local ───> API GitHub Copilot (terminal) (localhost:4141) (api.github.com) Format Anthropic Traduction Format OpenAI POST /v1/messages automatique POST /v1/chat/complétions Le proxy gère aussi : ● Authentification OAuth GitHub ● Renouvellement automatique du token ● Suppression du paramètre "thinking" ● Rate limiting

Flux d'authentification

Vous lancez le proxy
Le proxy demande un device code a GitHub
Vous entrez le code sur github.com/login/device
Le proxy recoit un token OAuth (ghu_...)

Sauvegarde localement, renouvele automatiquement.

A chaque requête, le token obtient un jeton temporaire Copilot

⚠ Consommation de tokens Copilot

IMPORTANT : Cette méthode consomme votre quota Copilot

Toutes les méthodes décrites dans ce guide font passer les requêtes de Claude Code a travers votre licence GitHub Copilot. Chaque interaction avec Claude Code consomme des tokens Copilot, exactement comme si vous utilisiez Copilot Chat, mais en beaucoup plus grande quantite.

Pourquoi Claude Code consomme autant ?

Claude Code fonctionne differemment d'un simple chat :

  • Contexte massif : a chaque requête, Claude Code envoie le contenu des fichiers ouverts, l'historique de conversation, les resultats d'outils, etc. Un seul message peut contenir 10 000 a 50 000 tokens de contexte.
  • Appels multiples : une seule commande utilisateur peut generer 3 a 15 appels API en cascade (lecture de fichiers, reflexion, edition, verification...).
  • Requetes en arrière-plan : sans les flags d'optimisation, Claude Code fait des appels supplémentaires pour les complétions, suggestions et télémétrie.
  • Reponses longues : les reponses contenant du code genere sont souvent 5 a 20x plus longues qu'un chat classique.

Ordres de grandeur

ActionTokens estimesEquivalent Copilot Chat
Question simple ("qu'est-ce que ce fichier fait ?")5 000 - 15 000~3-5 messages chat
Commande d'edition ("corrige le bug ligne 42")50 000 - 150 000~30-50 messages chat
Tache complexe ("ajoute une API REST avec tests")200 000 - 500 000~100-200 messages chat
Session de 30 minutes500 000 - 2 000 0001 journee entiere de chat
Session de travail d'une journee5 000 000+Quota hebdomadaire

Comment reduire la consommation

1. Variables d'environnement obligatoires

# Réduit les appels en arrière-plan (complétions, suggestions)
DISABLE_NON_ESSENTIAL_MODEL_CALLS=1

# Désactive le trafic non-essentiel (télémétrie, checks)
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
Sans ces variables, la consommation double ou triple

Claude Code fait des dizaines d'appels supplémentaires que vous ne voyez pas. Ces variables les desactivent.

2. Rate limiting sur le proxy

# Minimum 30 secondes entre chaque requête
npx copilot-api@latest start --rate-limit 30 --wait

3. Désactiver la recherche web

# Dans settings.json
"permissions": { "deny": ["WebSearch"] }

4. Choisir des modèles économes

ANTHROPIC_MODEL=gpt-4.1-mini          # au lieu de gpt-4.1
ANTHROPIC_SMALL_FAST_MODEL=gpt-4.1-mini

5. Bonnes pratiques d'utilisation

  • Soyez precis : "corrige le bug ligne 42 de server.js" consomme moins que "corrige les bugs"
  • Utilisez /compact dans Claude Code pour compresser le contexte
  • Fermez et relancez Claude Code regulierement pour reinitialiser le contexte
  • Surveillez le quota : npx copilot-api@latest check-usage
  • Évitez les "agent loops" ou Claude Code enchaine les outils sans fin

6. Surveillance du quota

# Voir la consommation actuelle
npx copilot-api@latest check-usage

# Voir les stats en temps reel (si le proxy tourne)
curl http://localhost:4141/usage

Méthode 1B : copilot-proxy avec daemon

Facile npm/npx — github.com/Jer-y/copilot-proxy

npx @jer-y/copilot-proxy@latest start --claude-code

Mode daemon (arrière-plan)

# Installer en global
npm install -g @jer-y/copilot-proxy

# Authentification
copilot-proxy auth

# Demarrer en daemon
copilot-proxy start -d

# Gestion
copilot-proxy status       # Voir le statut
copilot-proxy logs         # Voir les logs
copilot-proxy stop         # Arrêter
copilot-proxy restart      # Redémarrer
copilot-proxy enable       # Auto-démarrage au boot
copilot-proxy disable      # Désactiver l'auto-démarrage
Auto-démarrage au boot

copilot-proxy enable cree selon l'OS :

  • Linux : service systemd dans ~/.config/systemd/user/
  • macOS : plist dans ~/Library/LaunchAgents/
  • Windows : tache planifiee dans le Task Scheduler

Resultat : le proxy tourne en permanence. Vous n'avez plus qu'a lancer claude.

Méthode 2C : LiteLLM (Python)

Moyen pip — Le plus flexible, interface web d'admin.

Installation

pip install 'litellm[proxy]'
litellm --version

Configuration

Créez copilot-config.yaml :

model_list:
  - model_name: claude-sonnet-4
    litellm_params:
      model: github_copilot/claude-sonnet-4
      extra_headers:
        editor-version: "vscode/1.85.1"
        Copilot-Integration-Id: "vscode-chat"

  - model_name: gpt-4.1
    litellm_params:
      model: github_copilot/gpt-4.1
      extra_headers:
        editor-version: "vscode/1.85.1"
        Copilot-Integration-Id: "vscode-chat"

  - model_name: gpt-4.1-mini
    litellm_params:
      model: github_copilot/gpt-4.1-mini
      extra_headers:
        editor-version: "vscode/1.85.1"
        Copilot-Integration-Id: "vscode-chat"

litellm_settings:
  drop_params: true    # OBLIGATOIRE - supprime "thinking"
drop_params: true est obligatoire

Sans cette ligne, le paramètre thinking envoyé par Claude Code provoque une erreur 400 sur l'API Copilot.

Lancement

# Obtenir un token (si pas déjà fait)
npx copilot-api@latest auth --show-token
export GITHUB_TOKEN=ghu_xxxxxxxxxxxx

# Lancer LiteLLM
litellm --config copilot-config.yaml --port 4000

settings.json pour LiteLLM

{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:4000",
    "ANTHROPIC_AUTH_TOKEN": "sk-litellm-static-key",
    "ANTHROPIC_MODEL": "github_copilot/claude-sonnet-4",
    "ANTHROPIC_SMALL_FAST_MODEL": "github_copilot/gpt-4.1-mini",
    "DISABLE_NON_ESSENTIAL_MODEL_CALLS": "1",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  }
}

Dashboard web

LiteLLM inclut une interface d'administration sur http://localhost:4000/ui : requêtes en temps reel, consommation de tokens, gestion des modèles, erreurs.

Sécurité et risques

API non-officielle

Tous ces proxies utilisent des API reverse-engineered de GitHub Copilot. Ce n'est PAS supporté officiellement.

  • Risque de suspension temporaire en cas d'abus détecté
  • Les API peuvent changer sans préavis
  • Pas de support officiel

Bonnes pratiques

  • Ne commitez jamais vos tokens : ajoutez .claude-copilot-auth.json et .copilot-api/ a .gitignore
  • Proxy en local uniquement : n'exposez jamais sur 0.0.0.0
  • Utilisez le rate-limiting pour eviter la detection d'abus
  • Consultez votre equipe sécurité avant utilisation en entreprise

Tokens & Quota : comprendre la consommation

Pourquoi la consommation est si elevee

Claude Code n'est pas un simple chatbot. A chaque interaction, il :

  1. Envoie tout le contexte (fichiers, historique, resultats d'outils) = 10 000-50 000 tokens d'entrée
  2. Peut enchainer 3 a 15 appels API en cascade pour une seule commande
  3. Genere des reponses longues avec du code complet = 2 000-10 000 tokens de sortie
  4. Fait des appels en arrière-plan (sans les flags d'optimisation)
